# 0006. 画像出力先のデフォルトをカレントディレクトリとし --output で上書き可能にする

- Date: 2026-06-10
- Status: Accepted

## Context

生成した画像の保存先をどのように決定するかを定める必要がある。
利便性（毎回指定しなくてよい）と柔軟性（任意のパスに保存できる）を両立させたい。

## Decision

画像の出力先はデフォルトでカレントディレクトリとし、`--output` フラグで上書き可能にする。

- デフォルト: `./output-YYYYMMDD-HHMMSS.png` のようなタイムスタンプ付きファイル名
- `--output <path>`: ファイルパスまたはディレクトリパスを指定可能

## Consequences

- 何も指定しなくても画像が保存されるためシンプルに使える
- スクリプトや自動化では `--output` で保存先を制御できる
- デフォルトのファイル名規則を決める必要がある
