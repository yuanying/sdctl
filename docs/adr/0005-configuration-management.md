# 0005. 設定管理に設定ファイルと環境変数を併用する（環境変数優先）

- Date: 2026-06-10
- Status: Accepted

## Context

Web UI の接続先 URL や認証情報などを毎回コマンドライン引数で指定するのは煩雑である。
一方で CI/CD 環境では設定ファイルよりも環境変数での管理が適している。

## Decision

設定管理に設定ファイルと環境変数を併用し、環境変数を優先する。

- 設定ファイルパス: `~/.config/sdctl/config.yaml`
- 環境変数プレフィックス: `SDCTL_` （例: `SDCTL_URL`, `SDCTL_API_KEY`）
- 優先順位: 環境変数 > 設定ファイル > デフォルト値

## Consequences

- 日常利用では設定ファイルに接続先を書いておくことで毎回指定が不要
- CI/CD や複数環境の切り替えは環境変数で対応できる
- 設定の読み込みロジックが設定ファイルのみの場合より複雑になる
