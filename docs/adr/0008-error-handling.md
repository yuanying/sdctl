# 0008. エラーハンドリングはシンプルなメッセージ表示のみとする

- Date: 2026-06-10
- Status: Accepted

## Context

API エラーや接続失敗時のエラー出力方針を決める必要がある。
詳細すぎると通常利用時にノイズになり、簡素すぎるとデバッグが困難になる。

## Decision

エラー発生時はシンプルなエラーメッセージのみを stderr に出力し、終了コードを非ゼロにする。

例:
```
error: connection refused (http://localhost:7860)
error: model not found: "v1-5-pruned"
```

## Consequences

- 通常利用時のエラー出力がノイズにならずシンプル
- スクリプトや CI での利用時は終了コードでエラー判定できる
- 詳細なデバッグ情報（スタックトレース等）は表示されないため、問題調査時は別途ログを確認する必要がある
