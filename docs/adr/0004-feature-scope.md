# 0004. 初期実装スコープを txt2img / img2img / モデル管理とする

- Date: 2026-06-10
- Status: Accepted

## Context

sdctl の初期バージョンで実装する機能の範囲を決める必要がある。
スコープが広すぎると実装が長期化し、狭すぎると実用性が低くなる。

## Decision

初期実装のスコープを以下の3機能とする：

- **txt2img**: テキストプロンプトから画像を生成
- **img2img**: 既存画像を入力に画像を生成
- **モデル管理**: 利用可能なモデルの一覧表示・切り替え

## Consequences

- CLI として一通りの Stable Diffusion 操作が完結する
- extras（高解像度化）や PNG info 取得などは初期スコープ外となる
- 3機能を cobra のサブコマンドとして定義することでコマンド体系が明確になる
