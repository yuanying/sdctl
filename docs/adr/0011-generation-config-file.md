# 0011. 画像生成パラメーターと プロンプトをYAMLファイルで指定できるようにする

- Date: 2026-06-11
- Status: Accepted

## Context

画像生成のパラメーター（ステップ数、CFGスケール、サンプラー等）やプロンプトを毎回CLIフラグで指定するのは煩雑であり、設定の再利用や共有が難しい。
特定のスタイルや品質設定を繰り返し使う場合、ファイルとして保存・管理できることが望ましい。

## Decision

`txt2img` / `img2img` コマンドに `--params` および `--prompt` フラグを追加し、YAMLファイルで設定を指定できるようにする。

`--config` はルートコマンドの永続フラグ（WebUI接続設定）として既に使用されているため、生成パラメーター用フラグには `--params` を採用する。

### ファイル構成

- **パラメーターファイル**（`--params`）: 生成パラメーター全般とデフォルトのネガティブプロンプトを記述
- **プロンプトファイル**（`--prompt`）: ポジティブプロンプトを記述。ネガティブプロンプトも記述可能で、その場合はパラメーターファイルのデフォルト値を上書きする

### 優先順位

CLIフラグ > プロンプトファイル > パラメーターファイル > デフォルト値

### その他の仕様

- ファイルフォーマット: YAML（コメント記述可能）
- ファイル指定: CLIフラグのみ（デフォルトファイルの自動読み込みなし）
- バッチ生成: 非対応（`--prompt` は1ファイルのみ）
- 対応スコープ: txt2img・img2img 両対応
- 必須パラメーターが未指定の場合はエラー終了し、不足項目を明示する

## Consequences

- プロンプトやパラメーターをファイルで管理・再利用できる
- スタイルや品質設定をパラメーターファイルとして共有できる
- ネガティブプロンプトのデフォルト値をパラメーターファイルに集約しつつ、プロンプトごとの上書きも可能
- CLIフラグが常に優先されるため、ファイルをベースにその場での調整が容易
- デフォルトファイルの自動読み込みがないため、ファイルを使わない場合の挙動は従来通り
